# Release Checklist — VKarmani Desktop

> Цель: стандартизировать выпуск релиза и снизить риск регрессий для desktop-клиента на Tauri + React + Rust.

## Как использовать

- Отмечайте пункты только после фактической проверки.
- В каждом разделе фиксируйте ссылку на артефакт: PR, лог сборки, скрин, релизную задачу.
- Если пункт не применим, укажите причину (`N/A + объяснение`).

## Метаданные релиза

- Версия: `v__________`
- Ветка релиза: `release/v__________`
- Ответственный за релиз: `@__________`
- Дата/время старта (UTC): `____-__-__ __:__`
- Ссылка на релизный PR: `https://...`
- Ссылка на задачу/эпик: `https://...`

---

## 0) Подготовка ветки и Git (блокер)

- [ ] Ветка релиза создана от актуальной `main`/`work`.
- [ ] Настроен `origin` (`git remote -v` не пустой).
- [ ] Открыт PR из release-ветки; назначены ревьюеры.
- [ ] В PR есть: scope, риски, rollback-план.
- [ ] Добавлен тег релиза (`vX.Y.Z`) после финального approve.
- [ ] `git status` чистый перед публикацией.

**Артефакт/ссылка:** `______________________________`

## 1) Версии и артефакты

- [ ] Версия в `package.json` актуальна.
- [ ] Версия в `src-tauri/Cargo.toml` синхронизирована.
- [ ] Проверено наличие runtime-артефактов Windows:
  - [ ] `resources/core/windows/xray.exe`
  - [ ] `resources/core/windows/geoip.dat`
  - [ ] `resources/core/windows/geosite.dat`
  - [ ] `resources/core/windows/wintun.dll`
- [ ] Проверены контрольные суммы (если используется процесс checksum/signing).

**Артефакт/ссылка:** `______________________________`

## 2) Сборка и базовые проверки

- [ ] Установлены зависимости: `npm ci`.
- [ ] Frontend build проходит: `npm run build`.
- [ ] Tauri build проходит: `npx tauri build`.
- [ ] Сборка воспроизводится на чистом окружении (новая VM/чистая машина).
- [ ] Проверен запуск собранного приложения после установки.

**Артефакт/ссылка:** `______________________________`

## 3) Smoke-тест критических сценариев

- [ ] Авторизация (ключ / UUID / subscription URL).
- [ ] Синхронизация профиля и серверов.
- [ ] Подключение в режиме **Proxy**.
- [ ] Подключение в режиме **TUN**.
- [ ] Диагностика runtime и сети.
- [ ] Поведение при потере сети.
- [ ] Корректное отключение и повторное подключение.

**Артефакт/ссылка:** `______________________________`

## 4) Updater / обновления

- [ ] Успешное обновление с предыдущей версии.
- [ ] Негативный сценарий: недоступный/повреждённый пакет.
- [ ] Приложение корректно переживает ошибку обновления (без «кирпича»).
- [ ] После обновления приложение запускается, профиль и настройки читаются.
- [ ] Проверен rollback или безопасное восстановление.

**Артефакт/ссылка:** `______________________________`

## 5) Безопасность и данные

- [ ] Логи не содержат секретов/токенов/подписок.
- [ ] Временные файлы/конфиги runtime обрабатываются корректно.
- [ ] Проверено хранение пользовательских параметров и профиля.
- [ ] Проверена обработка сетевых/TLS ошибок.
- [ ] Проверены права доступа к файлам и каталогам runtime.

**Артефакт/ссылка:** `______________________________`

## 6) Релизная документация

- [ ] Подготовлены **Release Notes** (новое, исправления, breaking changes).
- [ ] Описаны Known Issues и ограничения.
- [ ] Добавлены инструкции поддержки (диагностика, сбор логов, FAQ).
- [ ] Обновлены пользовательские инструкции при изменениях UX.

**Артефакт/ссылка:** `______________________________`

- [ ] Загружены релизные артефакты.
- [ ] Проверена целостность/подписи (если применимо).
- [ ] Обновлены каналы доставки релиза.
- [ ] Анонс релиза подготовлен и опубликован.
- [ ] Поддержка уведомлена о времени релиза и known issues.

**Артефакт/ссылка:** `______________________________`

## 8) Post-release мониторинг (24–48 часов)

- [ ] Мониторинг первых ошибок/крашей.
- [ ] Мониторинг updater-метрик и фидбека пользователей.
- [ ] Подготовлен и подтверждён hotfix-план.
- [ ] Зафиксирован итог: релиз стабильный / нужен hotfix.

**Артефакт/ссылка:** `______________________________`

---

## Быстрый Go/No-Go (финальное решение)

Публикуем релиз только если все пункты ниже отмечены:

- [ ] Сборка проходит локально и на чистой машине.
- [ ] Критические smoke-тесты зелёные.
- [ ] Updater протестирован в success/failure сценариях.
- [ ] Release Notes готовы и согласованы.
- [ ] Rollback-план проверен и понятен дежурной команде.
